EMBEDDED SOFTWARE DEVELOPER
AUTOMATED TELLER MACHINE

General Overview

I collaborated with 3 of my Mechatronics Engineering colleagues on this project. This project spanned for just over 2 months starting from the initial design to the final presentation day. This is a prime example of a true Mechatronics project as it incorporated mechanical, electrical and software design. We used a Lego Mindstorms NXT 2.0 micro-controller kit and we coded all the functionality in RobotC (a variation of C). This was a great project in terms of learning how to code for a real life physical application. I personally gained a lot of embedded software experience and learned how to deal with limited memory on the micro-controller. I also gained valuable experience in many other software design concepts such as error-checking principles, developing safe and secure code, real-time dynamic calculating and complex debugging techniques.

Hardware

The mechanical design of this ATM was relatively simple as this was a prototype and not meant to be used in the real world. For example, the real final product would need to be fully covered with durable material to protect it from unauthorized people breaking into the ATM and stealing money. Our prototype does not incorporate this outer "shield" but rather displays the internal parts of an ATM. We integrated three motors into this design. The motor responsible for dispensing money had controlled movement (motor encoder + timers). We also used a color sensor for reading the client's card and a touch sensor for detecting when a user swipes a client card.

Software

On the software side of things we used object oriented programming practices and separated all of our code into functional blocks. This code is fairly complex as we truly pushed the Lego Mindstorm NXT micro-controller to it's maximum in terms of GUI and processor speed. You can view the code on my GitHub.
